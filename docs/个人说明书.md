### 👋我期待你如何和我协作

|                |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **工作方式**       | 聪明工作比努力工作、甚至比高效工作更重要 - 时刻想清楚与公司、leader和主要协同方对齐了吗？<br><br>真正的对齐不是机械性理解需求方的任务（What to do），而是要先了解Why（上下文，为什么做），再想What（做什么），最后才是How（怎么做），大量bad case是顶层没对齐就去执行具体问题，很有可能对齐以后发现整个事都不该做                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **和我沟通**       | 因为消息多，大部分群我默认不@我都不看，需要我处理/回复的事，辛苦在群里@我<br><br>收到我的消息，不用回复1或OK，在表情上打OK表情就行，信息降噪<br><br>和我1：1提前一天写好[和Grace 1：1 模板](https://realapp.feishu.cn/docs/doccnTST3QCiQxFNK3lfRn3e08F) 发给我，我们提前在文档预沟通划线评论，再当面聊                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **给我提需求**      | 如果1分钟内我没办法马上处理完给反馈的事，麻烦帮我加飞书的任务（待办）并根据优先级加截止日期<br><br>我📅日历空的时间都可以预约开会，最好提前1天，希望提前在日程中附上模板和会前沟通文档，文档写清上下文和议题                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **我提需求给你**     | 希望你能及时主动和我反馈/更新进展，不需要我来问「怎么样了」<br><br>- 有延期也尽量在发现风险的时候第一时间说，不等到最后一刻我问了才说<br>    <br>- 如果有哪里卡住了，你推不动，不管是团队内部还是公司外部，都可以第一时间来找我求助，我来推<br>    <br>- 我们的节奏和业务是变化很快的，及时对齐和同步非常重要 - 更多职场素养参考我写的[年薪翻倍的工作习惯.pdf](https://meta.feishu.cn/file/boxcnNvKryRyJngEqxdOl58Zjye)                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **你遇到困难**      | 欢迎随时找我求助，把我当做是你的资源；无需大小事都请示我的意见，如果你拿不准，我希望你让我做选择题而不是填空题，沟通时提出你的观察和建议的选项<br><br>- 因为我相信在具体的事上，你的上下文比我更多，你比我更专业，你的思考也更多，我的快思考不应该要比你的慢思考来得更深入，我只是可能经验、信息、资源更多一些<br>    <br>- 我认为独立思考、决策、在实践中假设检验，包括试错是最快让你成长的方式                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **同步/汇报工作/提问** | 先总结再展开，结论和需要我做什么说在开头，主动带上预计完成时间，尽量简短有逻辑，降低占用我的管理带宽<br><br>- 实际工作中，希望少一些模糊主观「我觉得」，多一些客观分析数据后，**理性归因**告诉我事实，比如过去3个月，找了N个候选人，其中M人（X%）因为xxx原因拒绝了我们，调研了市场这个岗位的情况，建议针对xxx原因这么改进 - 1... 2... 3...<br>    <br>- 怎么知道这是不是一个高效沟通/汇报？如果沟通时我总要追问很多问题，那可能需要反思这些问题是想了没说，还是根本没想，这些问题在下次类似沟通中是需要主动同步的。比如我常问这些问题是因为<br>    <br>    - **需要我做什么决策？**- 一些同学打了很多字，但一眼看过去并不知道只是同步信息，还是需要我定什么；有没有选项，比如有A、B两个选项，你推荐哪个？<br>        <br>    - **你的建议呢？**- 在你希望我做决策的时候，没有带上你的建议方案<br>        <br>    - **一共多少钱？**- 说了很多细节/局部费用，但总成本并没有说<br>        <br>    - **预计什么时候能完成？**- 时间是重要变量，通常告诉我截止日期之后，只要你之前不频繁延期，我都不会在截止日期之前问你<br>        <br><br>总结：换位思考如果你是leader/老板/合作方，你希望对方和你怎么沟通。想提升结构化思考、沟通和写作的同学可以去看「金字塔原理」 |
| **提审批**        | 算大帐再算小账，先说一共增加了多少钱（单月/全年/该周期内），不要先说一堆细节，钱不多的话往往不需要说那么多<br><br>审单价预算的时候只看局部绝对价格或折扣，很容易觉得贵，但如果做了单月/全年几种情况的预估演算，很可能会容易通过很多                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **你和我想法不一样**   | **我的意见/建议也不一定对，如果你认为不合理，可以直接提出来**，比如<br><br>- 产品需求不应来自于我/CEO/产品经理的感觉，应该来源于对用户的洞察、理解和数据分析，功能/策略的好坏，应该是用户说的算<br>    <br>- 公司政策也不应是我拍脑袋定，而是听用户（员工）的想法是什么，怎么样能把福利给到对员工和对组织真正有益的地方<br>    <br><br>**一定不能唯上，不能对leader/老板有特殊待遇；不同的想法要当面说，一定不能当面答应，背后不执行或差执行**                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **你和同事的习惯不一样** | **公司/团队同事过去的习惯也不一定对，任何事情应该去思考本质问题和本质解是什么**<br><br>- 不要说「过去我们都是这样」，过去经验≠最佳实践，不应按按惯性行动<br>    <br>- 很多人讨论问题时，喜欢说：对不起，这是「公司」的规定，公司总是最后的背锅侠。没有一个人叫公司，每件事都必须落实到具体的人。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
